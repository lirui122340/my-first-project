# 省钱旅游小程序

## 一、项目概述

一款面向大学生及预算敏感型旅行者的微信小程序，核心目标是帮助用户以最低成本规划旅行。首期核心功能为：**输入出发城市，一键获取所有可直达城市的剩余火车票**，用户从结果中选择心仪目的地，后续逐步扩展更多省钱出行功能。

---

## 二、核心功能

### 2.1 火车票直达查询（已实现）

核心交互流程：**输入出发城市 → 查看所有可直达城市及余票 → 选择目的地**

#### 用户操作流程

```

        │
        ▼

        │
        ▼
┌─────────────────────────────┐
│   可直达城市列表（按余票/价格排序）  │
│                               │
│  🏙 上海    余票 126  最低 ¥553 │
│  🏙 杭州    余票 89   最低 ¥264 │
│  🏙 南京    余票 203  最低 ¥132 │
│  🏙 广州    余票 45   最低 ¥862 │
│  ...                          │
└─────────────────────────────┘
        │  点击某个城市
        ▼
┌─────────────────────────────┐
│   该城市所有车次详情列表         │
│                               │
│  G1  06:36→11:48  二等座 ¥553 │

│  G7  09:00→13:28  二等座 ¥553 │
│  ...                          │

```

#### 已实现功能

- ✅ 用户输入出发城市和出发日期，无需指定目的地
- ✅ 系统自动查询从该城市出发所有可直达城市的余票情况
- ✅ 以城市为维度聚合展示：目的城市名称、总剩余票数、最低票价、车次数量
- ✅ 点击目的城市后展开所有车次详情（车次、时间、余票、票价）
- ✅ 支持按余票数量、最低价格排序
- ✅ 支持车次类型筛选（高铁动车/普通列车）
- ✅ 票价智能估算（内置热门线路真实票价 + 基于车次类型的费率估算）
- ✅ 41个热门城市映射，覆盖全国主要城市
- ✅ 页面滚动浏览，支持大量城市数据展示

### 3.3 数据获取方案

项目采用**双数据源**策略，优先使用 12306 官方接口，聚合数据作为备用：

#### 主数据源：12306 官方接口

```
1. 查询余票：
   https://kyfw.12306.cn/otn/leftTicket/queryZ
   ?leftTicketDTO.train_date=2026-05-01
   &leftTicketDTO.from_station=BJP
   &leftTicketDTO.to_station=SHH
   &purpose_codes=ADULT

2. 车站名映射：
   https://kyfw.12306.cn/otn/resources/js/framework/station_name.js

3. 多端点尝试：queryZ → queryA → queryG → query（自动切换）
```

#### 备用数据源：聚合数据 API

```
https://apis.juhe.cn/fapigw/train/query
?start=北京&end=上海&date=2026-05-01&key=YOUR_KEY
```

#### 票价估算策略

由于 12306 票价接口有反爬保护，采用"已知价格表 + 智能估价"策略：

1. **内置热门线路真实票价**：北京↔上海、北京↔广州等 23 条热门线路
2. **基于车次类型的费率估算**：根据 G/D/C/Z/T/K 不同车次类型的官方费率 × 估算里程
3. **反向线路匹配**：A→B 无价格时尝试查找 B→A 的价格

## 五、API 接口文档

```
GET /api/ticket/destinations

请求参数：
  from_city    string  出发城市（如"北京"）
  date         string  出发日期（如"2026-05-01"）
  sort_by      string  排序方式（可选，seats-按余票数/price-按最低价，默认 seats）

响应示例：
{
  "code": 0,
  "message": "success",
  "data": {
    "fromCity": "北京",
    "date": "2026-05-01",
    "destinations": [
      {
        "city": "南京",
        "totalSeats": 203,
        "minPrice": 132,
        "trainCount": 58
      },
      {
        "city": "上海",
        "totalSeats": 126,
        "minPrice": 553,
        "trainCount": 45
      }
    ]
  }
}
```

## 九、注意事项

1. **合规性**：12306 数据仅供学习使用，不得用于商业倒票等违法行为
2. **请求频率**：后端并发数 ≤ 5，请求间隔 ≥ 800ms，避免对 12306 服务器造成压力
3. **数据缓存**：相同查询条件在 5 分钟内返回缓存数据，减少重复请求
4. **错误处理**：网络异常、接口变更等情况需有友好的用户提示
5. **隐私安全**：不收集用户个人敏感信息，API Key 等密钥不可提交到代码仓库
6. **反爬处理**：12306 接口需携带有效 Cookie 和 User-Agent，多端点自动切换
