# 省钱旅游小程序

## 一、项目概述

一款面向大学生及预算敏感型旅行者的微信小程序，核心目标是帮助用户以最低成本规划旅行。首期核心功能为：**输入出发城市，一键获取所有可直达城市的剩余火车票**，用户从结果中选择心仪目的地，后续逐步扩展更多省钱出行功能。

---

## 二、核心功能

### 2.1 火车票直达查询（首期重点）

核心交互流程：**输入出发城市 → 查看所有可直达城市及余票 → 选择目的地**

#### 用户操作流程

```
输入出发城市 + 出发日期
        │
        ▼
  后端批量查询 12306 余票
        │
        ▼
┌─────────────────────────────┐
│   可直达城市列表（按余票/价格排序）  │
│                               │
│  🏙 上海    余票 126  最低 ¥553 │
│  🏙 杭州    余票 89   最低 ¥264 │
│  🏙 南京    余票 203  最低 ¥132 │
│  🏙 广州    余票 45   最低 ¥862 │
│  ...                          │
└─────────────────────────────┘
        │  点击某个城市
        ▼
┌─────────────────────────────┐
│   该城市所有车次详情列表         │
│                               │
│  G1  06:36→11:48  二等座 ¥553 │
│  G3  07:00→12:18  二等座 ¥553 │
│  G7  09:00→13:28  二等座 ¥553 │
│  ...                          │
└─────────────────────────────┘
```

#### 功能要点

- 用户仅需输入**出发城市**和**出发日期**，无需指定目的地
- 系统自动查询从该城市出发所有可直达城市的余票情况，以城市为维度聚合展示：
  - 目的城市名称
  - 该城市总剩余票数（各车次余票之和）
  - 最低票价（帮助用户快速判断性价比）
- 用户点击某个目的城市后，展开该城市所有车次详情：
  - 车次、出发/到达时间、历时
  - 各席别（硬座/硬卧/软卧/二等座/一等座）余票数量
  - 票价
- 支持按余票数量、最低价格排序
- 支持筛选：车次类型（高铁/动车/普通）、出发时段

### 3.3 12306 数据获取方案

> **重要提示**：12306 没有官方公开 API，需通过以下方式获取数据。
> 由于本项目的核心流程是"输入出发城市→获取所有可直达城市余票"，后端需要**批量并发查询**多个目的城市的余票数据，然后按城市聚合返回。

#### 方案 A：第三方 API 服务（推荐入门）

使用聚合数据、极速数据、APISpace 等平台提供的 12306 查询接口：

- 优点：接入简单、稳定、合规
- 缺点：有调用次数限制，高级功能需付费
- 适合：项目初期快速验证功能

**批量查询策略**：后端维护一份"热门直达城市映射表"，当用户输入出发城市时，并发请求该城市对应的多个目的城市：

```
请求示例（以 APISpace 为例，后端并发调用多次）：

用户输入：出发城市=北京，日期=2026-05-01

后端并发请求：
  GET .../ticket12306/query?from=北京&to=上海&date=2026-05-01
  GET .../ticket12306/query?from=北京&to=杭州&date=2026-05-01
  GET .../ticket12306/query?from=北京&to=南京&date=2026-05-01
  GET .../ticket12306/query?from=北京&to=广州&date=2026-05-01
  ...（根据城市映射表决定查询哪些城市）

后端聚合后返回：
{
  "code": 0,
  "data": {
    "fromCity": "北京",
    "date": "2026-05-01",
    "destinations": [
      {
        "city": "上海",
        "totalSeats": 126,
        "minPrice": 553,
        "trainCount": 45
      },
      {
        "city": "杭州",
        "totalSeats": 89,
        "minPrice": 264,
        "trainCount": 32
      }
    ]
  }
}
```

#### 方案 B：自建爬虫代理

通过 Node.js 后端模拟 12306 网站请求：

- 优点：无调用次数限制、数据完整
- 缺点：需处理反爬、验证码、接口变更维护成本高
- 适合：有一定开发经验后采用

**关键接口**：

```
1. 查询余票（需指定出发站和到达站代码）：
   https://kyfw.12306.cn/otn/leftTicket/query
   ?leftTicketDTO.train_date=2026-05-01
   &leftTicketDTO.from_station=BJP
   &leftTicketDTO.to_station=SHH
   &purpose_codes=ADULT

2. 车站名映射（获取城市→站点代码）：
   https://kyfw.12306.cn/otn/resources/js/framework/station_name.js
```

**批量查询实现要点**：

```
出发城市 → 查询所有可直达城市的流程：

1. 加载城市站点映射（station_name.js → 本地数据库）
2. 根据出发城市，获取该城市所有站点代码（如北京→BJP/BXP/VNP）
3. 从本地"直达城市映射表"中获取该城市可直达的所有目的城市
4. 使用 Promise.all 并发查询各目的城市的余票
5. 按目的城市聚合结果：统计总余票数、最低票价、车次数量
6. 写入 Redis 缓存（key: from_city + date, TTL: 5分钟）
```

**直达城市映射表构建方式**：

```
方式一：预置热门线路（推荐初期使用）
  维护一份 JSON 配置，手动录入主要城市间的直达线路：
  {
    "北京": ["上海", "杭州", "南京", "广州", "深圳", "成都", ...],
    "上海": ["北京", "杭州", "南京", "广州", ...],
    ...
  }

方式二：从 12306 车次数据自动构建（进阶）
  定时爬取全国车次时刻表，解析出每个出发站的到达站列表，
  构建完整的直达城市关系图，存入数据库。
```

**注意事项**：
- 必须设置合理的请求间隔（建议并发数 ≤ 5，单次间隔 ≥ 500ms），避免被封 IP
- 需携带有效的 Cookie 和 User-Agent
- 接口可能随时变更，需做好容错处理
- 仅用于学习目的，商业使用需获得授权

## 七、注意事项

1. **合规性**：12306 数据仅供学习使用，不得用于商业倒票等违法行为
2. **请求频率**：后端必须做请求限流，避免对 12306 服务器造成压力
3. **数据缓存**：相同查询条件在短时间内应返回缓存数据，减少重复请求
4. **错误处理**：网络异常、接口变更等情况需有友好的用户提示
5. **隐私安全**：不收集用户个人敏感信息，API Key 等密钥不可提交到代码仓库
