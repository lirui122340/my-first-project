# 逐行讲解（面向小白）

这份文档的目标：你不用懂“架构/设计模式”，也能跟着代码一步步看懂它在做什么，然后能自己改出新功能。

---

## 先说一个最重要的点：为什么要用 `python src/main.py` 启动？

本项目的包在 `src/game/` 下（例如 `from game.config import ...`）。

- 当你用 `python src/main.py` 运行时，Python 会把 `src/` 放进模块搜索路径，所以能找到 `game/` 这个包。
- 如果你在项目根目录直接执行 `python -c "import game"`，一般会失败，因为这时搜索路径里没有 `src/`。

所以**新手阶段你只需要记住**：

- **用 `python src/main.py` 启动即可。**

## 入口文件：`src/main.py` 逐行讲解

下面先贴出完整入口文件，后面按行解释。

```1:73:D:\github.file\tank\src\main.py
from __future__ import annotations

import sys

import pygame

from game.config import (
    BG_COLOR,
    FPS,
    SCREEN_HEIGHT,
    SCREEN_TITLE,
    SCREEN_WIDTH,
)
from game.scenes.gameover import GameOverScene
from game.scenes.gameplay import GameplayScene
from game.scenes.menu import MenuScene


class SceneManager:
    def __init__(self) -> None:
        self._scene = MenuScene(self)

    def switch(self, scene_name: str, payload: dict | None = None) -> None:
        payload = payload or {}
        if scene_name == "menu":
            self._scene = MenuScene(self)
        elif scene_name == "gameplay":
            self._scene = GameplayScene(self, **payload)
        elif scene_name == "gameover":
            self._scene = GameOverScene(self, **payload)
        else:
            raise ValueError(f"Unknown scene: {scene_name}")

    def handle_event(self, event: pygame.event.Event) -> None:
        self._scene.handle_event(event)

    def update(self, dt: float) -> None:
        self._scene.update(dt)

    def render(self, screen: pygame.Surface) -> None:
        self._scene.render(screen)


def main() -> int:
    pygame.init()
    pygame.display.set_caption(SCREEN_TITLE)
    screen = pygame.display.set_mode((SCREEN_WIDTH, SCREEN_HEIGHT))
    clock = pygame.time.Clock()

    manager = SceneManager()

    running = True
    while running:
        dt = clock.tick(FPS) / 1000.0
        for event in pygame.event.get():
            if event.type == pygame.QUIT:
                running = False
                break
            manager.handle_event(event)

        manager.update(dt)
        screen.fill(BG_COLOR)
        manager.render(screen)
        pygame.display.flip()

    pygame.quit()
    return 0


if __name__ == "__main__":
    raise SystemExit(main())
```

### 第 1 行：`from __future__ import annotations`

这一行的作用：让 Python 把类型标注（例如 `dict | None`）用更“宽松”的方式处理，减少某些版本下的类型问题。  
你可以先把它理解成：“**类型写起来更顺手**，对游戏运行没影响”。

### 第 3 行：`import sys`

这里目前其实**没用到**（我早期习惯性会先写上）。  
你现在可以先忽略；后续如果要做“命令行参数（比如选择关卡）”会用得到。

### 第 5 行：`import pygame`

导入 pygame：它负责 **开窗口、画图、读键盘、播音效**。

### 第 7–16 行：从 `game.*` 导入配置和场景

你可以把它理解成两类：

- **配置**：窗口大小、FPS、颜色等（在 `game/config.py`）
- **场景**：菜单、游戏中、结算（在 `game/scenes/`）

### 第 19–41 行：`SceneManager`（场景管理器）

你可以把“场景”当成不同页面：

- 菜单页面（Menu）
- 游戏页面（Gameplay）
- 结算页面（GameOver）

`SceneManager` 只做一件事：**当前是哪一页，就把输入/更新/渲染都交给那一页处理。**

- **第 20–21 行**：一开始就进入菜单 `MenuScene`
- **第 23–32 行**：`switch(...)` 切换页面  
  `payload` 是“带参数切换”，例如进入游戏时带上 `level_path`
- **第 34–41 行**：把 `handle_event / update / render` 转发给当前场景

### 第 44–67 行：`main()`（游戏主循环）

这段是 pygame 游戏的标准骨架：

- **第 45 行**：初始化 pygame
- **第 46–48 行**：设置窗口标题、窗口大小、时钟（用来控制 FPS）
- **第 50 行**：创建场景管理器（默认在菜单）
- **第 52–65 行**：循环直到退出

重点看两块：

#### A. 计算 dt（第 54 行）

`dt` 是“这一帧经过了多少秒”，例如 1/60 秒左右。  
我们用它来实现“不同电脑速度不同，但移动速度一致”：

- 每帧移动距离 \(= 速度 \times dt\)

#### B. 事件处理（第 55–59 行）

pygame 会把键盘/鼠标/窗口关闭等都放进事件队列，你用 `pygame.event.get()` 逐个取出来。

- 遇到 `pygame.QUIT`（点了窗口 X）就退出
- 其他事件交给当前场景处理（比如菜单里按 Enter 开始）

#### C. 更新 + 渲染（第 61–64 行）

顺序固定：

1. `update(dt)`：更新位置、AI、子弹等“逻辑”
2. `screen.fill(...)`：清屏（不清屏会“拖影”）
3. `render(...)`：画出这一帧应该出现的东西
4. `flip()`：把这一帧显示出来

### 第 70–71 行：脚本入口

只有当你直接运行这个文件时，才会执行 `main()`。  
`raise SystemExit(...)` 是更规范的退出方式（返回退出码）。

## 游戏主场景：`GameplayScene`（你玩的时候大部分逻辑都在这）

你先记住：`GameplayScene` 做的事情就是：

- 读输入（玩家移动、按 Space 开火）
- 更新敌人 AI
- 更新子弹与碰撞
- 判断胜利/失败并切换到结算场景

下面贴出 `GameplayScene` 里最关键的几个函数，并解释它们干什么。

### 1）初始化：加载关卡、生成玩家/敌人（`__init__`）

```30:52:D:\github.file\tank\src\game\scenes\gameplay.py
        self.level = load_level(level_path)
        self.player = spawn_player(self.level)

        self.enemy_ctrl = {id(e): EnemyController(ai=SimpleEnemyAI()) for e in self.enemies}

```

- **`load_level(...)`**：读取 `levels/level_01.json`，拿到“墙、出生点”等数据
- **`spawn_player/spawn_enemies`**：根据出生点生成坦克（坦克就是一个 `pygame.Rect` + 一些属性）
- **`enemy_ctrl`**：给每个敌人配一个 AI（目前是最简单的“随机巡逻 + 追玩家”）
- **`_solid_rects`**：所有“不能穿过去的东西”（墙 + 屏幕边界），碰撞时用

### 2）每帧更新：先更新玩家→敌人→子弹→胜负（`update`）

```84:97:D:\github.file\tank\src\game\scenes\gameplay.py
    def update(self, dt: float) -> None:
        if self.paused:
            return

        self.time_s += dt
        self.player.update_timers(dt)
        for e in self.enemies:
            e.update_timers(dt)

        self._update_player(dt)
        self._update_enemies(dt)
        self._update_bullets(dt)
        self._resolve_end_conditions()
```

你可以把它理解成“固定流水线”：

1. 暂停时不更新
2. 计时（用于结算界面显示用时）
3. 更新一些计时器（比如开火冷却）
4. 玩家更新
5. 敌人更新
6. 子弹更新 + 碰撞
7. 判断胜/败
